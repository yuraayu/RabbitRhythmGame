# **（改訂版）うさぎが人参を食べるリズムゲーム 企画書**

## **1\. ゲームの概要**

| 項目 | 内容 |
| :---- | :---- |
| **タイトル（仮）** | うさぎが人参を食べるリズムゲーム |
| **ジャンル** | リズムアクションゲーム（お手本模倣系） |
| **プラットフォーム** | PC (Unity) / モバイル (将来的な視野) |
| **ターゲットユーザー** | ・「リズム天国」のような、音とタイミングに集中するゲームが好きな人 ・シュールで可愛い世界観や、シンプルなゲームを求める人 |
| **コンセプト** | お手本のリズムを記憶し、ウサギになりきって人参を完食する、コミカル・リズムアクション。 |

## **2\. ゲームシステム（詳細化）**

本作は「リズム天国」のように、\*\*「お手本フェーズ」**と**「プレイフェーズ」\*\*を繰り返して進行する。

### **2-1. ゲームの基本フロー**

1. **お手本フェーズ (Listen)**  
   * BGMに合わせて、机の上に人参がリズミカルに置かれていく。  
   * プレイヤーは「いつ」「どの順番で」置かれたかを**聴覚と視覚で記憶**する。  
   * （例：「タン、タン、タン」というリズムで人参が3本置かれる）  
   * プレイヤーはこのフェーズで入力することはできない。  
2. **プレイフェーズ (Play)**  
   * お手本が終わると、プレイヤーの入力受付が開始される。（例：「どうぞ！」の合図と「はい！」というSE）  
   * プレイヤーは、お手本と**全く同じリズム**で入力（スペースキー押下）する。  
   * 入力に合わせて、ウサギが人参をかじる（食べる）アクションを行う。  
   * 入力タイミングの正確さによって、判定が行われる。  
3. **判定・結果**  
   * 1つのシーケンス（お手本→プレイ）が終了すると、スコアとコンボが表示される。  
   * BGMが続く限り、次の「お手本フェーズ」が始まる。

### **2-2. ノーツ（人参）の種類**

ゲームの核となるのは「タイミング」であるため、ノーツの種類はシンプルに構成する。

* **通常ノーツ（人参）**  
  * お手本で1回置かれ、プレイで1回タップ（かじる）する、最も基本的なノーツ。  
* **（拡張案）長押しノーツ（太い人参）**  
  * お手本で「ポーー」と長い音が鳴り、プレイでも「ガリガリガリ…」とキーを押し続ける必要があるノーツ。  
* **（拡張案）連打ノーツ（小さい人参の束）**  
  * お手本で「タタタッ！」と置かれ、プレイでも「ｶﾞｶﾞｶﾞｯ\!」と連打が必要なノーツ。

### **2-3. 判定の種類（時間差ベース）**

プレイヤーの入力タイミングと、お手本の目標時間との「時間差（timeDiff）」で判定する。

| 判定 | 時間差 (秒) | スコア | コンボ |
| :---- | :---- | :---- | :---- |
| **Perfect** | ± 0.1秒 以内 | 300点 | 継続 (+1) |
| **Good** | ± 0.2秒 以内 | 150点 | 継続 (+1) |
| **Bad** | ± 0.3秒 以内 | 50点 | 継続 (+1) |
| **Miss** | 上記以外 / 入力忘れ | 0点 | **リセット (0)** |

### **2-4. スコアリング・コンボシステム**

* **スコア:** 各判定（Perfect, Good, Bad）の合計。  
* **コンボ:** 判定（Bad以上）を連続で成功させると加算される。  
* **コンボボーナス:** コンボ数に応じてスコアにボーナスが加算される。（例：基本スコア \+ (コンボ数 / 10 \* 50)）  
* **Miss**を出すとコンボは 0 にリセットされる。  
* **空振り:** 判定対象となるノーツがない（早すぎる/遅すぎる）タイミングでの入力は、Missにもならず、コンボもリセットされない（応相談）。

## **3\. サウンドデザイン案**

ゲームプレイの根幹をなすため、音の差別化が重要。

* **BGM:**  
  * フリー素材を使用。ゲームのシュールな雰囲気に合わせ、ローファイ・ヒップホップ、または気の抜けたコミカルなBGMが望ましい。  
  * BPM（テンポ）が一定で、リズムの土台として機能するもの。  
* **効果音 (SE):**  
  * **お手本（人参が置かれる音）:**  
    * 「**ポンッ**」や「**コン**」といった、乾いた音。リズムが聴き取りやすいよう、アタック感が強い音。  
  * **プレイヤー（ウサギがかじる音）:**  
    * 「**ガリッ！**」「**サクッ！**」といった、小気味良い咀嚼音。  
    * Perfect / Good / Bad で音色や音程を変えると、プレイヤーへのフィードバックが強化される。（例：Perfectは高音で「シャキーン！」、Badは低音で「モシャ…」）  
  * **UI効果音:**  
    * ゲームスタート、判定表示、コンボ表示時など。

## **4\. UI/UX の構成案**

「落下型」から「固定画面型」に変更する。

* **画面構成:**  
  * **中央上部 (80%):** ゲームプレイ領域。  
    * 中央に**ウサギ**（プレイヤーの分身）を配置。  
    * ウサギの前に\*\*机（テーブル）\*\*を配置。  
    * お手本フェーズ中、この机の上に人参が置かれる。  
  * **画面下部 (20%):** UI領域。  
    * スコア（Score）  
    * コンボ数（Combo）  
    * 判定表示（"Perfect\!"など）  
* **ゲーム進行の視覚的フィードバック:**  
  * **お手本フェーズ:** 画面上部に「**Listen\!**」や「**おぼえて！**」と表示。ウサギはじっと待機している。  
  * **プレイフェーズ:** 画面上部に「**Play\!**」や「**どうぞ！**」と表示。ウサギが入力待ちのポーズになる。  
  * **入力成功時:** ウサギが人参をかじるアニメーション。  
  * **入力失敗時:** ウサギが悲しそうな顔をする、または人参が消える。

## **5\. イラストの具体的な雰囲気**

* **テーマ:** 「シュールな可愛さ」「コミカルな誇張」  
* **ウサギ:**  
  * デフォルメされた、頭でっかちなデザイン。  
  * 表情は「人参への純粋な渇望」「食いしん坊」を表現。まん丸な瞳で、幸せそうに食べる。  
  * 怖さよりも、コミカルさ、愛嬌を優先する。  
* **人参（ノーツ）:**  
  * 少し不格好で、個性的なデザイン。完璧な三角形ではなく、少し曲がっていたり、葉が変な方向を向いていたりする。  
* **スタイル:**  
  * 太めで均一な線を使った、シンプルな線画。  
  * 白黒（または淡いグレー）を基調とし、ポップさとシュールさを両立させる。  
  * 背景は単色か、ざらついたテクスチャのみで構成し、ウサギと人参に集中させる。
